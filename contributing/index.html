    Руководство редактора¶

    Этот документ поможет вам писать красивые и структурированные статьи для Большой Энциклопедии NewEra и Эррландии. Вам не нужно знать HTML или программирование - всё, что нужно, это обычный текст и несколько простых правил.

    1. Правила оформления файлов¶

    Прежде чем начать писать, важно правильно сохранить файл (если вы работаете через редактор или GitHub).

    Где создавать¶

    Все статьи лежат в папке docs/. Просто положите свой .md файл в нужную подпапку:

    Раздел Путь
    Лор docs/lore/
    История docs/history/
    Организации docs/organisations/
    Личности docs/characters/
    Основные страницы docs/

    Магия меню¶

    Благодаря плагину awesome-pages, вам НЕ НУЖНО редактировать файлы конфигурации. Просто создайте файл в правильной папке, и он автоматически появится в меню сайта.

    Названия файлов¶

    Только латиница, нижний регистр, слова разделяются дефисом. Кириллица в именах файлов запрещена - это ломает ссылки.

    • ✅ docs/lore/ideology/ideology.md
    • ✅ docs/characters/eria/character.md
    • ❌ docs/Лор/Идеологии/Идеолгия.md
    • ❌ docs/characters/eria/Character.md

    Картинки¶

    Все изображения кладите в папку docs/images/. Не создавайте отдельные папки для картинок внутри папок со статьями.

    2. Основы Markdown¶

    Наш движок использует язык разметки Markdown. Вот самые частые элементы:

    Заголовки¶

    Используйте # для заголовков. Чем больше решёток, тем меньше заголовок.

    # Заголовок 1 (только один раз на странице - в самом верху!)
## Заголовок 2 (основные разделы статьи)
### Заголовок 3 (подразделы)
#### Заголовок 4 (редко используется)

    Форматирование текста¶

    • Жирный текст - для акцентов и важных терминов.
    • Курсив - для названий книг или мыслей.
    • ~~Зачёркнутый~~ - редко используется, но бывает полезен.
    • Код в строке - для выделения или специфичных терминов.

    Списки¶

    - Маркированный список (обычные точки)
  - Вложенный список (отступ в 2 пробела)
1. Нумерованный список
2. Второй пункт

    Ссылки¶

    [Текст ссылки](название-файла.md) - ссылка на другую статью вики.
[Внешняя ссылка](https://google.com) - ссылка на сайт в интернете.

    Совет: При ссылке на другую статью используйте относительный путь. Если статья лежит в папке lore/general, а вы ссылаетесь на characters/eria, пишите ../../characters/eria/character.md. ../ означают возврат на папку выше.

    3. Фишки нашего движка (Material)¶

    Наша Вики умеет больше, чем обычный текст. Используйте эти блоки, чтобы статьи выглядели лучше.

    Блоки-предупреждения (Admonitions)¶

    Идеально для лора, спойлеров и важных заметок.

    Как писать:

    !!! note "Заголовок блока"
    Текст внутри блока. Отступ в 4 пробела обязателен.

    Заголовок блока

    Текст внутри блока. Отступ в 4 пробела обязателен.

    Блок-спойлер (сворачивается)¶

    ??? warning "Нажми, чтобы узнать тайну"
    Здесь спрятан текст. Читатель должен кликнуть, чтобы увидеть его.
    Нажми, чтобы узнать тайну

    Здесь спрятан текст. Читатель должен кликнуть, чтобы увидеть его.

    Типы блоков¶

    Тип Цвет Когда использовать
    note Синий Обычные заметки
    tip Зелёный Советы, подсказки
    warning Оранжевый Предупреждения
    danger Красный Опасность, важные ограничения
    quote Серый Цитаты, отсылки
    abstract Голубой Краткое содержание
    info Синий Дополнительная информация

    Вкладки (Tabs)¶

    Используйте, когда нужно показать альтернативы.

    === "Вкладка 1"
    Текст первой вкладки.

=== "Вкладка 2"
    Текст второй вкладки.

    Важно: После === и названия в кавычках нужно поставить два пробела перед текстом.

    Текст первой вкладки.

    Текст второй вкладки.

    Карточки (Cards)¶

    Отлично подходят для списков.

    <div class="grid cards" markdown>

-   :fontawesome-solid-user:{ .lg .middle } **Имя**
    ---
    Краткое описание.
    [Подробнее](название-статьи.md)

-   :fontawesome-solid-house:{ .lg .middle } **Название**
    ---
    Описание.
    [Подробнее](название-статьи.md)

</div>

    • Имя --- Краткое описание. Подробнее

    • Название --- Описание. Подробнее

    4. Работа с изображениями¶

    Простая вставка¶

    ![Описание картинки](../../images/my-image.jpg)

    Картинка с подписью и ограничением ширины¶

    <figure markdown>
  ![Описание](../images/my-image.jpg){ width="500" loading=lazy }
  <figcaption>Подпись под картинкой)</figcaption>
</figure>

    Атрибут loading=lazy ускоряет загрузку страницы. Всегда добавляйте его.

    Переключение картинок по теме¶

    Наша вики поддерживает светлую и тёмную темы. Если у вас есть разные версии картинки для каждой темы, используйте классы theme-light и theme-dark:

    ![Светлая версия](../images/logo-light.png){ .theme-light }
![Тёмная версия](../images/logo-dark.png){ .theme-dark }

    Движок автоматически покажет нужную версию в зависимости от выбранной темы.

    Сетка картинок (Галерея)¶

    <div class="grid" markdown>

![Картинка 1](../images/photo1.jpg)
![Картинка 2](../images/photo2.jpg)
![Картинка 3](../images/photo3.jpg)

</div>

    5. Скрытые настройки (Frontmatter)¶

    В самом верху каждого .md файла есть блок между тремя дефисами ---. Это настройки страницы.

    Базовый frontmatter¶

    ---
title: "Название во вкладке браузера"
description: "Краткое описание для поисковиков"
---

# Заголовок статьи

Текст статьи...

    Скрытие элементов¶

    Если вы делаете красивую главную страницу или страницу с огромной картой, меню может мешать. Добавьте параметр hide:

    ---
hide:
  - navigation  # Скрыть левое боковое меню
  - toc         # Скрыть правое оглавление
  - footer      # Скрыть подвал сайта
---

    Учитывайте, что без navigation левое меню исчезнет.

    6. Лицензия и авторство¶

    Вся Большая Энциклопедия NewEra и Эррландии распространяется по лицензии CC BY-SA 4.0.

    Что это значит для авторов:¶

    ✅ Вы сохраняете авторство - ваше имя будет указано в истории Git
    ✅ Вы можете редактировать чужие статьи - это вики, она общая
    ✅ Материалы могут использоваться другими - при условии указания источника

    Правила цитирования:¶

    • Если вы используете текст из другого источника (книга, статья, другой сайт) - укажите источник в конце статьи.
    • Если вы адаптируете чужой материал - укажите оригинал и автора.
    • Не копируйте материалы, защищённые авторским правом, без разрешения.

    Пример указания источника:¶

    ---
title: "История Эррландии"
---

# История Эррландии

Текст статьи...

---

<small>

**Источники:** Адаптировано из книги "История Эррландии" (Автор, дата).  
**Автор статьи:** [ТвойНик](https://github.com/твой-ник)

</small>

    7. Редактируем и создаём статьи через браузер (самый простой способ)¶

    Вам не нужно устанавливать Git или разбираться в командной строке. Всё делается прямо в вашем браузере, а все изменения попадают к нам на проверку через Pull Request (это просто "предложение изменений", которое мы рассмотрим и, если всё хорошо, примем).

    Редактирование существующей статьи¶

    1. Откройте любую страницу вики, которую хотите исправить.
    2. В правом верхнем углу страницы нажмите значок ✏️ (Edit this page) - вы перейдёте на GitHub к исходному файлу.
    3. Внесите нужные правки в тексте.
    4. Используйте наш справочник по Markdown для форматирования.
    5. Если добавляете картинки - см. раздел Работа с изображениями.
    6. Прокрутите страницу вниз до блока Commit changes:
    7. В поле Commit message кратко опишите, что изменили (например, "Исправил опечатку в имени").
    8. Оставьте переключатель "Create a new branch" (по умолчанию).
    9. Нажмите зелёную кнопку "Propose changes".
    10. GitHub предложит создать Pull Request - нажмите зелёную кнопку "Create pull request".
    11. Готово. Ваше предложение появится в списке Pull Request, и модератор рассмотрит его в ближайшее время.

    Создание новой статьи¶

    1. Перейдите в нужную папку в репозитории.
    2. На главной странице репозитория нажмите docs/, затем выберите подпапку
    3. Вы также можете перейти напрямую по ссылке:
      https://github.com/northfern/NewEraWiki/tree/main/docs/НАЗВАНИЕ_ПАПКИ/НАЗВАНИЕ_ПОДПАПКИ
    4. Внутри нужной папки нажмите кнопку "Add file" → "Create new file".
    5. В поле Name your file введите имя файла строго по правилам:
    6. Только латиница, нижний регистр.
    7. Вместо пробелов - дефис -.
    8. Обязательно расширение .md.
      ✅ Правильно: errlandika.md, latonik.md.
      ❌ Неправильно: Эррландия.md, Латоник.md.
    9. В основном поле напишите текст статьи:
    10. Начните с frontmatter (заголовок и описание) - см. раздел Скрытые настройки.
    11. Затем заголовок первого уровня (# Название) и содержание.
    12. Используйте форматирование, вкладки, карточки и другие фишки из нашего руководства.
    13. Если вы хотите добавить картинки, сделайте это отдельно:
    14. Вернитесь на уровень docs/ и зайдите в папку images/.
    15. Нажмите "Add file" → "Upload files" и загрузите изображения.
    16. В статье ссылайтесь на них так: ../../images/ваша-картинка.jpg.
    17. После того как файл написан, спуститесь вниз к Commit changes:
    18. Напишите краткое сообщение (например, "Добавлена статья о Фрассии").
    19. Оставьте опцию "Create a new branch".
    20. Нажмите "Propose new file".
    21. Как и в случае с редактированием, появится предложение создать Pull Request - нажмите "Create pull request".

    Важные замечания¶

    • Все изменения проходят модерацию - мы проверим оформление, орфографию и синтаксис.
    • Если вы создаёте новую статью, не забудьте добавить ссылки на неё из других связанных статей, чтобы читатели могли легко переходить.
    • Если вы сомневаетесь в форматировании, посмотрите на уже существующие статьи как на пример.
    • Не удаляйте и не переименовывайте файлы через веб-интерфейс без согласования - лучше напишите нам в Discord.

    Что, если я хочу добавить несколько файлов за раз?¶

    Если вы правите много статей или добавляете целую коллекцию картинок, рекомендуем использовать GitHub Desktop или консоль. Подробные инструкции есть в официальной документации GitHub.

    8. Выбор редактора Markdown¶

    Если вы предпочитаете писать статьи офлайн, вот несколько удобных редакторов для разных систем:

    Редактор Платформы Особенности
    Typora Win / Mac / Linux WYSIWYG (скрывает разметку), платный ($15), есть триал
    Obsidian Win / Mac / Linux Бесплатный, мощная система ссылок, подходит для личных баз знаний
    MarkText Win / Mac / Linux Бесплатный, открытый, аналог Typora
    VS Code Win / Mac / Linux Бесплатный, требует установки расширений (Markdown All in One)
    ghostwriter Linux / Win Бесплатный, открытый, минималистичный
    MacDown Mac Бесплатный, популярный
    Apostrophe Linux Бесплатный, открытый, экосистема GNOME

    Для быстрых правок можно использовать любой текстовый редактор (Блокнот в Windows 11 уже поддерживает Markdown). Главное - сохранять файл с расширением .md и соблюдать наши правила оформления.

    9. Альтернативные способы отправки материалов¶

    Если вам неудобно работать с GitHub даже через браузер, вы можете отправить свою статью одним из следующих способов:

    Через Discord (для коротких заметок)¶

    Вы можете прислать текст с картинками в нашем Discord сервере или лично владельцу энциклопедии. Мы сами оформим его в Markdown и опубликуем.

    10. Что делать, если что-то сломалось?¶

    Текст не форматируется?¶

    Проверьте отступы. В Markdown пробелы имеют значение. Обычно это 4 пробела для вложенности.

    Картинка не отображается?¶

    Проверьте путь. Он должен начинаться с ../images/ и не содержать кириллицу.

    Ссылка ведёт в никуда?¶

    Убедитесб, что в конце ссылки стоит .md и путь написан правильно.

    • 🐙 GitHub: Создай Issue в нашем репозитории с описанием проблемы.
    • 💬 Discord: Напиши нам на сервере
