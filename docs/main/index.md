# 📖 Руководство редактора

Привет! Этот документ поможет тебе писать красивые и структурированные статьи для **Большой Энциклопедии NewEra и Эррландии**. Тебе не нужно знать HTML или программирование — всё, что нужно, это обычный текст и несколько простых правил.

---

## 1. 📁 Правила оформления файлов

Прежде чем начать писать, важно правильно сохранить файл:

### Где создавать
Все статьи лежат в папке `docs/`. Просто положи свой `.md` файл в нужную подпапку:

| Раздел | Путь |
|--------|------|
| Лор (Эррландия, история, организации) | `docs/lore/` |
| Персонажи (NPC, игроки) | `docs/chars/` |
| Основные страницы | `docs/main/` |

### Магия меню
Благодаря плагину `awesome-pages`, тебе **НЕ НУЖНО** редактировать файлы конфигурации. Просто создай файл в правильной папке, и он автоматически появится в меню сайта!

### Названия файлов
Только **латиница, нижний регистр**, слова разделяются дефисом. Кириллица в именах файлов запрещена — это ломает ссылки.

- ✅ `docs/lore/ancient-magic.md`
- ✅ `docs/chars/northfern.md`
- ❌ `docs/Лор/Древняя Магия.md`
- ❌ `docs/chars/NorthFern.md`

### Картинки
Все изображения клади в папку `docs/images/`. Не создавай отдельные папки для картинок внутри папок со статьями.

## 2. ️ Основы Markdown

Наш движок использует язык разметки **Markdown**. Вот самые частые элементы:

### Заголовки

Используй `#` для заголовков. Чем больше решёток, тем меньше заголовок.

```markdown
# Заголовок 1 (только один раз на странице — в самом верху!)
## Заголовок 2 (основные разделы статьи)
### Заголовок 3 (подразделы)
#### Заголовок 4 (редко используется)
```

### Форматирование текста

- **Жирный текст** — для акцентов и важных терминов.
- *Курсив* — для названий книг, заклинаний или мыслей.
- ~~Зачёркнутый~~ — редко используется, но бывает полезен.
- `Код в строке` — для выделения игровых команд или специфичных терминов.

### Списки

```markdown
- Маркированный список (обычные точки)
  - Вложенный список (отступ в 2 пробела)
1. Нумерованный список
2. Второй пункт
```

### Ссылки

```markdown
[Текст ссылки](название-файла.md) — ссылка на другую статью вики.
[Внешняя ссылка](https://google.com) — ссылка на сайт в интернете.
```

💡 **Совет:** При ссылке на другую статью используй относительный путь. Если статья лежит в папке `lore`, а ты ссылаешься на `chars`, пиши `../chars/players.md`.

## 3.  Фишки нашего движка (Material)

Наша Вики умеет больше, чем обычный текст. Используй эти блоки, чтобы статьи выглядели профессионально.

### Блоки-предупреждения (Admonitions)

Идеально для лора, спойлеров и важных заметок.

**Как писать:**

```markdown
!!! note "Заголовок блока"
    Текст внутри блока. Отступ в 4 пробела обязателен!
```

**Как выглядит:**

!!! note "Заголовок блока"
    Текст внутри блока. Отступ в 4 пробела обязателен!

### Блок-спойлер (сворачивается)

```markdown
??? warning "Нажми, чтобы узнать тайну"
    Здесь спрятан текст. Читатель должен кликнуть, чтобы увидеть его.
```

??? warning "Нажми, чтобы узнать тайну"
    Здесь спрятан текст. Читатель должен кликнуть, чтобы увидеть его.

### Типы блоков

| Тип | Цвет | Когда использовать |
|-----|------|-------------------|
| `note` | 🔵 Синий | Обычные заметки |
| `tip` | 🟢 Зелёный | Советы, подсказки |
| `warning` | 🟠 Оранжевый | Предупреждения |
| `danger` | 🔴 Красный | Опасность, важные ограничения |
| `quote` |  Серый | Цитаты, отсылки |
| `abstract` |  Голубой | Краткое содержание |
| `info` |  Синий | Дополнительная информация |

### Вкладки (Tabs)

Используй, когда нужно показать альтернативы (например, точки зрения разных фракций на одно событие).

```markdown
=== "Вкладка 1"
    Текст первой вкладки.

=== "Вкладка 2"
    Текст второй вкладки.
```

⚠️ **Важно:** После `===` и названия в кавычках нужно поставить **два пробела** перед текстом.

=== "Вкладка 1"
    Текст первой вкладки.

=== "Вкладка 2"
    Текст второй вкладки.

### Карточки (Cards)

Отлично подходят для списков персонажей, локаций или предметов.

```markdown
<div class="grid cards" markdown>

-   :fontawesome-solid-user:{ .lg .middle } **Имя Персонажа**
    ---
    Краткое описание персонажа или локации.
    [Подробнее](название-статьи.md)

-   :fontawesome-solid-house:{ .lg .middle } **Название Локации**
    ---
    Описание локации.
    [Подробнее](название-статьи.md)

</div>
```

<div class="grid cards" markdown>

-   :fontawesome-solid-user:{ .lg .middle } **Имя Персонажа**
    ---
    Краткое описание персонажа или локации.

## 4. 🖼️ Работа с изображениями

Картинки оживляют лор!

### Простая вставка

```markdown
![Описание картинки](../images/my-image.jpg)
```

### Картинка с подписью и ограничением ширины

```markdown
<figure markdown>
  ![Описание](../images/my-image.jpg){ width="500" loading=lazy }
  <figcaption>Подпись под картинкой (например, "Рисунок из дневника мага")</figcaption>
</figure>
```

💡 Атрибут `loading=lazy` ускоряет загрузку страницы. Всегда добавляй его!

### Переключение картинок по теме

Наша вики поддерживает светлую и тёмную темы. Если у тебя есть разные версии картинки для каждой темы, используй классы `theme-light` и `theme-dark`:

```markdown
![Светлая версия](../images/logo-light.png){ .theme-light }
![Тёмная версия](../images/logo-dark.png){ .theme-dark }
```

Движок автоматически покажет нужную версию в зависимости от выбранной темы.

### Сетка картинок (Галерея)

```markdown
<div class="grid" markdown>

![Картинка 1](../images/photo1.jpg)
![Картинка 2](../images/photo2.jpg)
![Картинка 3](../images/photo3.jpg)

</div>
```

## 5. 🔧 Скрытые настройки (Frontmatter)

В самом верху каждого `.md` файла есть блок между тремя дефисами `---`. Это настройки страницы.

### Базовый frontmatter

```markdown
---
title: "Название во вкладке браузера"
description: "Краткое описание для поисковиков"
---

# Заголовок статьи

Текст статьи...
```

### Скрытие элементов

Если ты делаешь красивую главную страницу или страницу с огромной картой, меню может мешать. Добавь параметр `hide`:

```markdown
---
hide:
  - navigation  # Скрыть левое боковое меню
  - toc         # Скрыть правое оглавление
  - footer      # Скрыть подвал сайта
---
```

## 6. ⚖️ Лицензия и авторство

Вся Большая Энциклопедия NewEra и Эррландии распространяется по лицензии **[CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/)**.

### Что это значит для авторов:

✅ **Ты сохраняешь авторство** — твоё имя будет указано в истории Git  
✅ **Ты можешь редактировать чужие статьи** — это вики, она общая  
✅ **Материалы могут использоваться другими** — при условии указания источника  

### Правила цитирования:

- Если ты используешь текст из другого источника (книга, статья, другой сайт) — **укажи источник** в конце статьи.
- Если ты адаптируешь чужой материал — укажи оригинал и автора.
- Не копируй материалы, защищённые авторским правом, без разрешения.

### Пример указания источника:

```markdown
---
title: "Древняя магия"
---

# Древняя магия

Текст статьи...

---

<small>

**Источники:** Адаптировано из книги "История Аркания" (Автор, 2024).  
**Автор статьи:** [ТвойНик](https://github.com/твой-ник)

</small>
```

## 7.  Как вносить вклад

### Через GitHub (рекомендуется)

1. **Форкни репозиторий** — нажми кнопку "Fork" на [странице репозитория](https://github.com/твой-ник/rp-wiki)

2. **Склонируй форк на свой компьютер:**
   ```bash
   git clone https://github.com/ТВОЙ-НИК/rp-wiki.git
   cd rp-wiki
   ```
   ⚠️ Замени `ТВОЙ-НИК` на свой никнейм на GitHub!

3. **Добавь основной репозиторий как upstream** (чтобы получать обновления):
   ```bash
   git remote add upstream https://github.com/твой-ник/rp-wiki.git
   ```

4. **Получи свежие изменения** (перед началом работы):
   ```bash
   git checkout main
   git pull upstream main
   ```

5. **Создай ветку** для своих изменений:
   ```bash
   git checkout -b my-new-article
   ```
   💡 Называй ветку осмысленно: `add-ancient-magic`, `fix-npc-stats`, `update-history`.

6. **Внеси изменения** и проверь, что всё правильно:
   ```bash
   git status
   ```

7. **Добавь все изменённые файлы** в коммит:
   ```bash
   git add .
   ```
   Или добавь конкретные файлы:
   ```bash
   git add docs/lore/my-article.md docs/images/my-image.jpg
   ```

8. **Сделай коммит** с понятным сообщением:
   ```bash
   git commit -m "Добавлена статья о древней магии"
   ```
   💡 Хорошие сообщения: `"Добавлена статья о..."`, `"Исправлены опечатки в..."`, `"Обновлены характеристики NPC..."`

9. **Запушь ветку** в свой форк:
   ```bash
   git push -u origin my-new-article
   ```

10. **Создай Pull Request** на GitHub:
    - Перейди в свой форк на GitHub
    - Нажми зелёную кнопку "Compare & pull request"
    - Опиши, что ты добавил или изменил
    - Нажми "Create pull request"

### Через веб-интерфейс GitHub (для небольших правок)

Если тебе нужно исправить опечатку или добавить короткую заметку, Git можно не устанавливать:

1. Открой нужный файл в [репозитории](https://github.com/твой-ник/rp-wiki)
2. Нажми иконку карандаша (✏️) в правом верхнем углу файла
3. Внеси изменения в редакторе
4. Прокрути вниз и нажми **"Commit changes..."**
5. Опиши изменения в поле "Commit message"
6. Выбери **"Create a new branch"** и нажми **"Propose changes"**
7. На следующей странице нажми **"Create pull request"**

## 8. 🆘 Что делать, если что-то сломалось?

### Текст не форматируется?
Проверь отступы. В Markdown отступы (пробелы) имеют значение. Обычно это **4 пробела** для вложенности.

### Картинка не отображается?
Проверь путь. Он должен начинаться с `../images/` и не содержать кириллицу.

### Ссылка ведёт в никуда?
Убедись, что в конце ссылки стоит `.md` и путь написан правильно.

### Переключатель тем не работает?
Очисти кэш браузера: нажми `Ctrl + Shift + R` (жёсткая перезагрузка).
